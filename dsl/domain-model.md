# 在庫管理ドメインモデル

関数型ドメインモデリングの考え方に基づくDSLで記述したドメインモデルです。

## 概要

- 最小管理単位は**在庫ロット**
- 販売単位は**販売案件**
- 販売案件に値段がつけられると**価格査定**
- 契約を締結する場合には査定を**販売契約**に紐づける

---

## ドメインモデル DSL

```
// ============================================================
// 在庫管理ドメイン
// ============================================================

// ----------------------------------------------------------------
// 基本値型
// ----------------------------------------------------------------

// 事業部コード
data 事業部コード = 整数

// 部門コード
data 部門コード = 整数

// 担当課コード
data 担当課コード = 整数

// ロット番号（年度・保管場所・連番の組み合わせで一意）
data ロット番号 = ロット番号年度 AND ロット番号保管場所 AND ロット番号連番

// 販売案件番号（年度・月度・連番の組み合わせで一意）
data 販売案件番号 = 販売案件番号年度 AND 販売案件番号月度 AND 販売案件番号連番

// 査定番号（年度・月度・連番の組み合わせで一意）
data 査定番号 = 査定番号年度 AND 査定番号月度 AND 査定番号連番

// 契約番号（年度・月度・連番の組み合わせで一意）
data 契約番号 = 契約番号年度 AND 契約番号月度 AND 契約番号連番

// 0以上、円単位
data 金額 = 整数

// 0.001以上
data 数量 = 数値

// 1以上
data 個数 = 整数

// ----------------------------------------------------------------
// ロット明細（在庫ロットの構成要素）
// ----------------------------------------------------------------

data ロット明細 = 品目区分
    AND 上位品区分?
    AND 商品分類コード
    AND 長さ規格下限
    AND 太さ規格下限
    AND 太さ規格上限
    AND 品質等級
    AND 個数
    AND 数量
    AND 検査結果区分?

// 一般品・上位品・特注品のいずれか
data 品目区分 = 一般品 OR 上位品 OR 特注品

// ----------------------------------------------------------------
// 在庫ロット（最小管理単位）
// ----------------------------------------------------------------

// 在庫ロットは製造の進捗に応じて状態が変わる
data 在庫ロット = 製造中ロット OR 製造完了ロット OR 変換指示済みロット OR 出荷指示済みロット OR 出荷完了ロット

data ロット共通 = ロット番号
    AND 事業部コード
    AND 部門コード
    AND 担当課コード
    AND 工程区分
    AND 検査区分
    AND 製造区分
    AND List<ロット明細> // 1件以上

// 工程区分：検査・製造・出荷等の工程の区分
data 工程区分 = 整数

// 検査区分：検査済み・未検査など
data 検査区分 = 整数

// 製造区分：製造方法の区分
data 製造区分 = 整数

data 製造中ロット = ロット共通

data 製造完了ロット = ロット共通
    AND 製造完了日

data 出荷指示済みロット = ロット共通
    AND 製造完了日
    AND 出荷期限日

data 出荷完了ロット = ロット共通
    AND 製造完了日
    AND 出荷期限日
    AND 出荷日

// 品目変換が指示された状態のロット
data 変換指示済みロット = ロット共通
    AND 製造完了日
    AND 変換先情報

// ----------------------------------------------------------------
// 販売案件（販売単位）
// ----------------------------------------------------------------

// 販売案件は直接・予約・委託の3種類がある
data 販売案件 = 直接販売案件 OR 予約販売案件 OR 委託販売案件

data 販売案件共通 = 販売案件番号
    AND 事業部コード
    AND 販売実施日
    AND List<在庫ロット> // 1件以上

// 直接販売案件は査定→契約のライフサイクルをたどる
data 直接販売案件 = 査定前直接販売案件
    OR 査定済み直接販売案件
    OR 契約済み直接販売案件
    OR 出荷指示済み直接販売案件
    OR 出荷完了直接販売案件

data 査定前直接販売案件 = 販売案件共通

data 査定済み直接販売案件 = 販売案件共通
    AND 価格査定

data 契約済み直接販売案件 = 販売案件共通
    AND 価格査定
    AND 販売契約

data 出荷指示済み直接販売案件 = 販売案件共通
    AND 価格査定
    AND 販売契約
    AND 出荷指示情報

data 出荷完了直接販売案件 = 販売案件共通
    AND 価格査定
    AND 販売契約
    AND 出荷指示情報
    AND 出荷完了日

// 予約販売案件は確定前後でライフサイクルが異なる
data 予約販売案件 = 仮予約案件
    OR 予約済み案件
    OR 予約確定済み案件
    OR 予約納品済み案件

data 仮予約案件 = 販売案件共通

data 予約済み案件 = 販売案件共通
    AND 予約価格

data 予約確定済み案件 = 販売案件共通
    AND 予約価格
    AND 予約確定日

data 予約納品済み案件 = 販売案件共通
    AND 予約価格
    AND 予約確定日
    AND 納品日

// 委託販売案件は委託業者に販売を委ねる
data 委託販売案件 = 委託指定前販売案件
    OR 委託指定済み販売案件
    OR 委託販売結果入力済み販売案件

data 委託指定前販売案件 = 販売案件共通

data 委託指定済み販売案件 = 販売案件共通
    AND 委託業者情報

data 委託販売結果入力済み販売案件 = 販売案件共通
    AND 委託業者情報
    AND 委託販売結果

// ----------------------------------------------------------------
// 価格査定(販売案件に値段をつける)
// ----------------------------------------------------------------

// 査定方式によって必要な情報が異なる
data 価格査定 = 通常査定 OR 顧客契約査定

data 査定共通 = 査定番号
    AND 査定日
    AND 納期
    AND 販売市場
    AND 基準単価適用日
    AND 期間調整率適用日
    AND 取引先調整率適用日
    AND 税抜予定総額
    AND List<ロット価格査定> // 1件以上

data 通常査定 = 査定共通

// 顧客契約査定は大口顧客との年間契約に基づく
data 顧客契約査定 = 査定共通
    AND 顧客契約番号
    AND 契約調整率

// ロットごとの価格査定情報
data ロット価格査定 = ロット番号
    AND 加工費?
    AND 個別受注加算?
    AND 等級加算?
    AND 予約加算?
    AND 調整率?
    AND 品質調整率?
    AND 製造費単価?
    AND 想定販売期間?
    AND 目標利益率?
    AND List<ロット明細価格査定> // 1件以上

// ロット明細ごとの価格査定情報
data ロット明細価格査定 = ロット明細
    AND 基準単価
    AND 期間調整率
    AND 取引先調整率
    AND 特例期間調整率?

// 予約価格は確定前後で不変条件が変わる
data 予約価格 = 暫定予約価格 OR 確定予約価格

data 予約価格共通 = 査定番号
    AND 査定日
    AND 予約対象ロット情報
    AND 予約金額

data 暫定予約価格 = 予約価格共通

data 確定予約価格 = 予約価格共通
    AND 確定日
    AND 確定金額

// ----------------------------------------------------------------
// 販売契約(査定に基づき締結する契約)
// ----------------------------------------------------------------

data 販売契約 = 契約番号
    AND 契約日
    AND 担当者
    AND 購入者
    AND 販売情報
    AND 販売価格情報
    AND 査定番号 // 紐づく価格査定

data 購入者 = 顧客番号
    AND 代理人氏名?

data 販売情報 = 販売種別
    AND 品目
    AND 納入方法
    AND 支払猶予条件?
    AND 販売方式
    AND 用途?
    AND 支払猶予金額?

// 販売方式：直販・代理店経由・契約販売など
data 販売方式 = 整数

// 販売種別：一般品・上位品・特注品など
data 販売種別 = 整数

data 販売価格情報 = 税抜契約金額
    AND 消費税額
    AND 税抜入金額
    AND 入金消費税額

// ----------------------------------------------------------------
// 出荷指示情報（出庫指示時に付与される）
// ----------------------------------------------------------------

// 出荷指示が行われた日付
data 出荷指示情報 = 出荷指示日

// ----------------------------------------------------------------
// 振る舞い
// ----------------------------------------------------------------

// 在庫ロットのライフサイクル
behavior 製造完了を指示する = 製造中ロット AND 製造完了日 -> 製造完了ロット OR 製造完了指示エラー
behavior 出荷を指示する = 製造完了ロット AND 出荷期限日 -> 出荷指示済みロット OR 出荷指示エラー
behavior 出荷完了を指示する = 出荷指示済みロット AND 出荷日 -> 出荷完了ロット OR 出荷完了指示エラー
behavior 製造完了を取り消す = 製造完了ロット -> 製造中ロット OR 取消エラー

// 販売案件のライフサイクル
behavior 販売案件を作成する = List<製造完了ロット> AND 販売案件種別 -> 販売案件 OR 販売案件作成エラー
behavior 販売案件を削除する = 査定前直接販売案件 -> 削除済み OR 削除エラー

// 価格査定のライフサイクル
behavior 価格査定を作成する = 査定前直接販売案件 AND 査定情報 -> 査定済み直接販売案件 OR 査定作成エラー
behavior 価格査定を更新する = 査定済み直接販売案件 AND 査定情報 -> 査定済み直接販売案件 OR 査定更新エラー
behavior 価格査定を削除する = 査定済み直接販売案件 -> 査定前直接販売案件 OR 査定削除エラー

// 販売契約のライフサイクル
behavior 販売契約を締結する = 査定済み直接販売案件 AND 契約情報 -> 契約済み直接販売案件 OR 契約締結エラー
behavior 販売契約を削除する = 契約済み直接販売案件 -> 査定済み直接販売案件 OR 契約削除エラー

// 出庫のライフサイクル（販売案件単位）
behavior 出庫を指示する = 契約済み直接販売案件 -> 出荷指示済み直接販売案件 OR 出庫指示エラー
behavior 出庫完了を指示する = 出荷指示済み直接販売案件 AND 出荷完了日 -> 出荷完了直接販売案件 OR 出庫完了指示エラー
behavior 出庫指示を取り消す = 出荷指示済み直接販売案件 -> 契約済み直接販売案件 OR 出庫指示取消エラー

// 予約のライフサイクル
behavior 予約価格を作成する = 仮予約案件 AND 予約価格情報 -> 予約済み案件 OR 予約価格作成エラー
behavior 予約を確定する = 予約済み案件 AND 確定日 -> 予約確定済み案件 OR 予約確定エラー
behavior 予約確定を取り消す = 予約確定済み案件 -> 予約済み案件 OR 予約確定取消エラー
behavior 納品を指示する = 予約確定済み案件 AND 納品日 -> 予約納品済み案件 OR 納品エラー

// 委託のライフサイクル
behavior 委託販売案件を指定する = 委託指定前販売案件 AND 委託業者情報 -> 委託指定済み販売案件 OR 委託指定エラー
behavior 委託販売案件指定を解除する = 委託指定済み販売案件 -> 委託指定前販売案件 OR 委託指定解除エラー
behavior 委託販売結果を入力する = 委託指定済み販売案件 AND 委託販売結果 -> 委託販売結果入力済み販売案件 OR 委託結果入力エラー

// 品目変換（ロットの品目区分変換）
behavior 品目変換を指示する = 製造完了ロット AND 変換先情報 -> 変換指示済みロット OR 変換指示エラー
behavior 品目変換指示を取り消す = 変換指示済みロット -> 製造完了ロット OR 変換指示取消エラー
```

---

## モデリングの判断ポイント

- **在庫ロット・販売案件の状態をORで分離**：状態ごとに必要なデータが変わるため、フラグではなく型で表現。例えば出荷完了ロットは出荷日が必須だが、製造中ロットには不要。

- **価格査定を通常査定/顧客契約査定に分離**：顧客契約査定は顧客契約番号と契約調整率が追加で必要なため、ORで分けた。

- **予約価格を確定前後で分離**：確定後は確定金額が必須となり不変条件が変わるため、状態として分離した。

- **販売案件の直接/予約/委託をORで完全分離**：それぞれライフサイクルが根本的に異なるため、フラグではなく型として表現。「予約販売案件に販売契約を締結する」といった業務上ありえない操作を型レベルで防げる。
