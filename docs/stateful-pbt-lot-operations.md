# ステートフルPBT: ロット状態遷移で検査すべき操作一覧

作成日: 2026-05-13

## 概要

`docs/stateful-pbt-fscheck-implementation-plan.md` に基づき、ロット状態遷移のステートフルPBTで検査すべき全操作を整理する。

## 対象状態

| モデル状態 | 実装型 |
|---|---|
| 製造中 | `ManufacturingLot` |
| 製造完了 | `ManufacturedLot` |
| 出荷指示済み | `ShippingInstructedLot` |
| 出荷完了 | `ShippedLot` |
| 変換指示済み | `ConversionInstructedLot` |

## 検査すべき操作

### 製造完了を指示する (`completeManufacturing`)

- 「製造中のロットに製造完了を指示する」に期待されるのは「成功（製造完了ロットになる）」
- 「すでに製造完了のロットに製造完了を指示する」に期待されるのは「不正遷移エラー」
- 「出荷指示済みのロットに製造完了を指示する」に期待されるのは「不正遷移エラー」
- 「出荷完了のロットに製造完了を指示する」に期待されるのは「不正遷移エラー」
- 「変換指示済みのロットに製造完了を指示する」に期待されるのは「不正遷移エラー」

### 製造完了を取り消す (`cancelManufacturingCompletion`)

- 「製造完了のロットの製造完了を取り消す」に期待されるのは「成功（製造中ロットに戻る）」
- 「製造中のロットの製造完了を取り消す」に期待されるのは「不正遷移エラー」
- 「出荷指示済みのロットの製造完了を取り消す」に期待されるのは「不正遷移エラー」
- 「出荷完了のロットの製造完了を取り消す」に期待されるのは「不正遷移エラー」
- 「変換指示済みのロットの製造完了を取り消す」に期待されるのは「不正遷移エラー」

## 横断的な性質（不変条件）

任意のコマンド列に対して常に成り立つべき性質:

- 「任意の成功した遷移の後」に期待されるのは「ロット共通情報（ロット番号・事業部コード・部門コード・担当課コード・工程区分・検査区分・製造区分・明細リスト）が変化しないこと」
- 「出荷完了に到達したロットに対する任意の操作」に期待されるのは「すべて不正遷移エラー（終端状態から戻れない）」
- 「取り消し操作が成功した場合」に期待されるのは「直前の状態に正確に戻ること（製造完了取消→製造中、変換指示取消→製造完了）」
- 「不正遷移エラーが返された場合」に期待されるのは「ロットの状態が一切変化しないこと」

## 状態遷移図（許可される遷移のみ）

```
製造中 ──[製造完了を指示する]──→ 製造完了
製造完了 ──[製造完了を取り消す]──→ 製造中
製造完了 ──[出荷を指示する]──→ 出荷指示済み
製造完了 ──[品目変換を指示する]──→ 変換指示済み
出荷指示済み ──[出荷完了を指示する]──→ 出荷完了（終端）
変換指示済み ──[品目変換指示を取り消す]──→ 製造完了
```

## ステートフルPBTでの検査方法

上記の個別ケース（30件）と横断的性質（4件）は、以下の形で一括検査する:

1. 初期状態「製造中」のロットを生成する
2. 6種のコマンドからランダムにコマンド列を生成する
3. 各ステップで、モデル（抽象状態）と実装（`InventoryLot`）に同じコマンドを適用する
4. モデルと実装の成功/失敗が一致することを確認する
5. 成功時は到達状態が一致することを確認する
6. 失敗時は状態が変化しないことを確認する
7. 全ステップでロット共通情報が維持されることを確認する
