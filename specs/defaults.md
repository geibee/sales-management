# 共通既定仕様（defaults）

依頼ごとに毎回書かなくてよい「このシステム全体の決まり」を一度だけ合意して置く場所。
各仕様テンプレートの該当欄は、**この既定から外れる場合のみ**理由付きで記入する。

- 既定の変更は影響が全画面・全 API に及ぶため、それ自体を 1 件の依頼（`00_change-request.md`）として扱う。
- 「要合意」印の項目は顧客との初回合意で確定させる。

| 項目 | 記入 |
|---|---|
| 状態 | **draft**（エンジニア起案の初期値。顧客合意で approved にする） |
| 顧客側承認者 / 承認日 | |

## 画面共通（DEF-UI）

| ID | 項目 | 既定 |
|---|---|---|
| DEF-UI-01 | 読み込み中 | 既存画面と同じ表示（スケルトン/スピナー） |
| DEF-UI-02 | データ 0 件 | 「〜はありません」形式の文言（固有文言は各ページ仕様） |
| DEF-UI-03 | 金額の書式 | 円・3 桁区切り・単位表示 |
| DEF-UI-04 | 日付の書式 | YYYY-MM-DD（JST） |
| DEF-UI-05 | 率の書式 | % 表記・小数第 1 位 |
| DEF-UI-06 | 項目エラー | 該当項目の直下にメッセージ表示 |
| DEF-UI-07 | 全体エラー | 既存の describeApiError 分類（通信・サーバ・権限）の文言 |
| DEF-UI-08 | 競合（他者が先に更新） | その旨を表示し再読み込みを促す。入力値は保持。自動マージしない |
| DEF-UI-09 | 権限がない操作 | ボタン非表示（「表示するが押せない」にする場合は各仕様に理由を書く） |
| DEF-UI-10 | 未ログイン | ログインへ誘導 |
| DEF-UI-11 | アクセシビリティ | キーボード操作可・フォーム項目にラベル・色だけで意味を伝えない・axe 検査適合 |
| DEF-UI-12 | 一覧のページング | 既存画面の方式に従う（既定件数・上限: **要合意**） |

## API 共通（DEF-API）

| ID | 項目 | 既定 |
|---|---|---|
| DEF-API-01 | エラー応答形式 | RFC 9457 ProblemDetails |
| DEF-API-02 | 更新系の競合制御 | 楽観ロック（version）。競合は 409 で拒否し、利用者が取り直して再操作 |
| DEF-API-03 | 認証・認可 | 未認証 401 / 権限不足 403 |
| DEF-API-04 | URL・命名 | 既存 `openapi.yaml` のリソース設計・命名に揃える（契約差分承認のレビュー観点） |

## 時刻・日付（DEF-TM）

| ID | 項目 | 既定 |
|---|---|---|
| DEF-TM-01 | タイムゾーン | JST 基準 |
| DEF-TM-02 | 「日」の数え方 | 暦日（営業日基準を使う場合は各仕様に明記） |

## バッチ・非同期の運用共通（DEF-OPS）

| ID | 項目 | 既定 |
|---|---|---|
| DEF-OPS-01 | 失敗の通知先 | **要合意**（チャネル・宛先） |
| DEF-OPS-02 | 実行記録 | 開始/終了時刻・処理件数・スキップ件数を必ず残す |
