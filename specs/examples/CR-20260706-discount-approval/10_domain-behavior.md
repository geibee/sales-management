## 2. 型スケッチ（エンジニア記入。`dsl/domain-model.md` 差分の下書き）

```text
// 直接販売案件に状態を 1 つ追加する
data 直接販売案件 =
  査定前直接販売案件
  OR 査定済み直接販売案件
  OR 値引き承認待ち案件      // ← 追加
  OR 契約済み直接販売案件

data 値引き承認待ち案件 = 査定済み直接販売案件の内容 AND 値引き申請

data 値引き申請 = 値引き率 AND 申請理由 AND 申請者

// 0 超〜10 以下、%、小数第 1 位まで（詳細は §5）
data 値引き率 = 数値

behavior 値引きを申請する =
  査定済み直接販売案件 AND 値引き申請 -> 値引き承認待ち案件 OR 値引き申請エラー

behavior 値引きを承認する =
  値引き承認待ち案件 AND 承認者 -> 査定済み直接販売案件 OR 値引き承認エラー

behavior 値引き申請を取り下げる =
  値引き承認待ち案件 AND 申請者 -> 査定済み直接販売案件 OR 値引き取り下げエラー
```
