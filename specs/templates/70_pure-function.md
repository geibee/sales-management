# 計算ロジック仕様（純粋関数）

> 適用対象: 金額・数量・率などの計算規則。バックエンド（F# workflow）かフロント（表示用換算）かは実装側が決める。
> **同じ計算を両側でやる場合はここに一本化して書き、双方のテストが同じ例表を使う。**

## メタ情報 ★

| 項目 | 記入 |
|---|---|
| 依頼 ID / 仕様 ID | CR-… / FN-01 |
| 計算の名前（業務語で） | 例: 値引き後契約額の算出 |

## 1. 入力と出力 ★

| 入力 | 型のイメージ | 単位・精度 |
|---|---|---|
| 査定額 | 金額 | 円（整数） |
| 値引き率 | 数値 | %・小数第 1 位まで |

| 出力 | 型のイメージ | 単位・精度 |
|---|---|---|
| 契約額 | 金額 | 円（整数） |

## 2. 計算規則 ★

数式または判定表で書く。日本語の散文だけにしない。

> 記入例: `契約額 = 査定額 × (100 − 値引き率) ÷ 100`、1 円未満は**切り捨て**。

## 3. 端数・特殊値の扱い ★（計算仕様の最重要欄）

| ケース | 扱い |
|---|---|
| 端数（1 円未満） | 切り捨て / 四捨五入 / 切り上げ ← 必ずどれか明記 |
| ゼロ（査定額 0 円、率 0%） | |
| 上限付近（金額の最大値） | 例: 業務上 10 億円を超えない前提でよい / 超えたらエラー |
| 計算不能（該当時: ゼロ除算等） | |

## 4. 常に成り立つ性質 ★（そのまま PBT の property になる。当てはまるものを書く）

- [ ] 出力は入力の範囲を出ない（例: 0 ≤ 契約額 ≤ 査定額）
- [ ] 単調性（例: 値引き率が大きいほど契約額は小さいか等しい）
- [ ] 合計の保存（例: 明細への按分合計 = 全体額。端数の帰属先: 最終明細に寄せる 等）
- [ ] 往復性（例: 率→額→率 で元に戻る。丸めがある場合は成り立たない旨を明記）
- [ ] 冪等性（例: 2 回適用しても 1 回と同じ）
- その他: 

## 5. 例・境界例 ★（顧客承認必須）

| ID | 入力 | 期待出力 | 備考 | 顧客承認 |
|---|---|---|---|---|
| FN-EX-01 | 査定額 100,000 円、率 8.0% | 92,000 円 | 代表例 | |
| FN-EX-02 | 査定額 99,999 円、率 0.1% | 99,899 円 | 端数切り捨ての確認（99,899.001 → 99,899） | |
| FN-EX-03 | 査定額 0 円、率 10.0% | 0 円 | ゼロ | |
