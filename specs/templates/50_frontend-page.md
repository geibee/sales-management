# フロントエンド ページ仕様

> 適用対象: 画面の追加、既存画面への項目・操作・導線の追加。
> コンポーネント分割・カスタム hook・ライブラリ選定は書かない（AI が Atomic Design 規約とゲートの範囲で決める）。
> 入力値の業務ルールは `10_domain-behavior.md` を参照する。画面独自の即時チェック（入力補助）だけをここに書く。

## 1. 目的と導線 ★

| 項目 | 記入 |
|---|---|
| 誰が・何をしに来るページか | 例: 営業担当が査定済み案件に値引きを申請する |
| どこから来るか | 例: 販売案件詳細の操作ボタン / サイドバー |
| 完了後どこへ行くか | 例: 同じ詳細画面に留まり、状態表示が更新される |
| 一覧 / 詳細 / 作成 / ダイアログ の別 | |

## 2. 表示項目 ★

| 項目 | 意味・出所 | 表示形式の要望 | 並び・絞り込み |
|---|---|---|---|
| 案件番号 | | | 既定の並び: 申請日の新しい順 |
| 想定契約額 | 査定額 −値引き | 円・3 桁区切り | |

- 件数が多いときの扱い（ページング/上限）: 
- **金額・日付・率の表示形式**は既存画面の規約に従う（変えたい場合のみ理由付きで記載）

## 3. 入力項目 ★（入力を伴う画面のみ）

| 項目 | 必須 | 業務ルール（DA 参照） | 画面での入力補助 | 初期値 |
|---|---|---|---|---|
| 値引き率 | ○ | DA-01 §5 | %単位で数値入力。範囲外は送信前に項目エラー表示 | 空 |
| 申請理由 | ○ | 500 文字以内 | 残り文字数表示 | 空 |

- 未保存で画面を離れるときの扱い: 確認する / しない

## 4. 状態 × 操作可否マトリクス ★（操作ボタンがある画面は必須。全セルを埋める）

| ドメイン状態 \ 操作 | 値引き申請 | 承認 | 取り下げ |
|---|---|---|---|
| 査定済み | 表示・押せる | 非表示 | 非表示 |
| 値引き承認待ち | 非表示 | 表示（上長のみ） | 表示（申請者のみ） |
| 契約済み | 非表示 | 非表示 | 非表示 |

- 本表に書くのは**状態による違い**。ロールによる出し分けの正は DA の権限マトリクス（§7）で、
  ここには「（上長のみ）」のように注記だけする（矛盾したら DA が正）。
- 「非表示」か「表示するが押せない（理由をツールチップ等で示す）」かを区別して書く（既定は非表示: DEF-UI-09）。

## 5. 画面上のふるまい（既定から外れる点と画面固有の文言のみ ★）

読み込み中・0 件・項目/全体エラー・競合・権限なし・未ログインの挙動は既定（`defaults.md` DEF-UI-01〜10）に従う。
ここには**画面固有の文言と、既定から外れる点だけ**を書く（逸脱には理由必須）。

| 場面 | 固有文言・既定との差分 |
|---|---|
| データ 0 件の文言 | 例: 「承認待ちの申請はありません」 |
| 操作成功時 | 例: トースト表示 + 状態バッジが「承認待ち」に変わる |
| （既定からの逸脱があれば追加） | 理由: |
